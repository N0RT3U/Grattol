# Grattol 예측 모델 개선 설명서

## 0. 한 줄 요약

기존 Holt-Winters 모델이 잘 안 맞았던 가장 큰 이유는 **너무 오래된 급등 구간까지 똑같이 믿고 학습했기 때문**이다.  
그래서 나는 **최근 56일만 사용**하고, 그 안에서도 **유독 튀는 1일(2020-01-27)**만 살짝 눌러 준 뒤, **주간 패턴을 배우는 ETS(=Holt-Winters 계열)**로 다시 학습했다.  
그 결과, 마지막 30일 테스트에서 성능이 아래처럼 개선됐다.

| 구분 | 기존 노트북 Holt-Winters | 최종 모델 |
| --- | ---: | ---: |
| R2 | -0.6207 | **0.2311** |
| R2(log1p) | -0.5832 | **0.2645** |
| RMSE | 898.63 | **618.97** |
| MAPE | 13.18% | **8.56%** |

즉,

- R2는 `0.8518p` 개선
- R2(log1p)는 `0.8477p` 개선
- RMSE는 `31.12%` 감소
- MAPE는 `4.6166%p` 감소 (`35.04%` 개선)

## 1. 이번 작업의 목표

목표는 딱 2개였다.

1. **설명력 높이기**
   모델이 실제 데이터의 움직임을 더 잘 설명하게 만들기
2. **예측력 높이기**
   실제 미래 30일을 더 정확히 맞히게 만들기

여기서 설명력은 `R2`, `R2(log1p)`로 봤고, 예측력은 `RMSE`, `MAPE`, `MAE`로 봤다.

중학생 버전으로 말하면:

- `R2`는 "그래프의 굴곡을 얼마나 잘 따라가나?"
- `RMSE`는 "평균적으로 얼마나 크게 틀리나?"
- `MAPE`는 "실제값 대비 몇 % 정도 틀리나?"

## 3. 왜 기존 Holt-Winters가 잘 안 맞았나

핵심 원인은 3개다.

### 3-1. 너무 긴 기간을 한 번에 학습했다

기존 모델은 **2019-10-01부터 2020-01-30까지 전체 학습 구간**을 거의 한 덩어리로 봤다.

문제는:

- 11월 말은 매우 뜨거운 기간
- 12월은 식는 기간
- 1월은 다시 올라가는 기간
- 2월은 상대적으로 안정된 기간

즉, **과거의 분위기가 계속 바뀌는데도**, 모델은 이걸 하나의 규칙으로 평균 내 버렸다.

중학생식 비유:

> 시험 점수를 예측하려는데,  
> 작년 중간고사 대폭락, 기말고사 폭등, 이번 모의고사 안정권 점수를  
> 전부 똑같이 섞어 평균 내면,  
> "지금 상태"를 제대로 맞히기 어렵다.

### 3-2. 이상치(급등일)의 영향이 너무 컸다

예를 들어 `2019-11-22`는 `15,278`건이다.  
그런데 2020-02 테스트 구간 평균은 `5,783.17`건이다.

즉,

- 11월 급등치는 2월 평균보다 약 `2.64배` 큼

이런 값이 그대로 들어가면 모델이 **평소보다 과하게 큰 예측**을 내놓기 쉽다.

### 3-3. 최근 패턴보다 오래된 패턴을 더 많이 섞었다

테스트 구간은 `2020-01-31 ~ 2020-02-29`였다.  
이때는 사실상 "최근 한두 달의 리듬"이 가장 중요하다.

그런데 전체기간 모델은 10월, 11월의 오래된 패턴도 많이 반영했다.  
그래서 **최근 분위기 맞춤형 모델**이 필요했다.

## 4. 이번에 무엇을 했는가: MECE하게 정리

내가 한 일은 크게 4단계다.

### 4-1. 데이터 구조를 다시 확인했다

먼저 확인한 사실:

1. 목표는 `Grattol 일별 event_count`
2. 데이터는 `152일`이라 아주 길지 않다
3. 주간 패턴(요일 효과)은 있을 가능성이 높다
4. 11월 말, 1월 말 급등이 존재한다
5. 최근 구간과 과거 구간의 수준이 다르다

### 4-2. 비교할 모델 후보를 여러 개 만들었다

총 **43개 후보**를 돌렸다.

구성은 다음과 같다.

1. 기존 전체기간 Holt-Winters 계열
2. 전체기간 + 완만한 이상치 캡핑
3. 최근 구간만 보는 모델
4. 최근 구간 + 이상치 캡핑
5. 같은 최근 구간에서도 `window 길이`, `cap 비율`, `damped 여부`를 바꾼 조합

즉, "모델 종류"만 바꾼 것이 아니라,

- 얼마나 최근 데이터만 볼지
- 이상치를 얼마나 누를지
- 추세를 완만하게 꺾을지

를 같이 실험했다.

### 4-3. 모델 선택 기준을 숫자로 고정했다

모델을 감으로 고르지 않고 아래 기준으로 골랐다.

#### 1) 설명력

- `R2`
- `R2(log1p)`

#### 2) 예측 오차

- `RMSE`
- `MAPE`

#### 3) 최종 선택 점수

최종 선택은 다음 공식을 썼다.

`selection_score = 0.6 * R2(log1p) + 0.4 * R2`

이렇게 한 이유:

- `R2(log1p)`는 급등일이 있어도 상대적으로 안정적
- `R2`는 원래 스케일에서 실제 맞춤 정도를 보여줌

즉,

- 로그 기준 설명력도 보고
- 실제 원래 숫자 기준 설명력도 같이 보자

라는 뜻이다.

### 4-4. 마지막 30일 홀드아웃으로 같은 조건에서 비교했다

테스트 구간:

- `2020-01-31 ~ 2020-02-29`

학습 구간:

- 그 이전 날짜들

즉, 미래를 미리 보지 않고 진짜 예측하듯 평가했다.

## 5. 어떤 모델이 최종 1등이었나

최종 1등 모델:

- 이름: `hw_recent_w56_q0985_damped`
- 뜻:
  - `recent_w56`: 최근 56일만 학습
  - `q0985`: 상위 1.5%만 캡핑
  - `damped`: 추세가 무한히 커지지 않게 완만하게 꺾음

### 5-1. 정확한 설정

| 항목 | 값 |
| --- | --- |
| 학습 창(window) | 최근 56일 |
| 모델 | additive ETS |
| 계절성 | 주간(7일) |
| 추세 | additive |
| damped trend | 사용 |
| 이상치 처리 | `q = 0.985` 상단 캡핑 |

### 5-2. 캡핑은 실제로 얼마나 일어났나

캡핑 값:

- `9161.775`

최근 56일 학습 구간에서 이 값보다 큰 날은 딱 **1일**뿐이었다.

| 날짜 | 원래 값 | 캡핑 후 |
| --- | ---: | ---: |
| 2020-01-27 | 10,047 | 9,161.775 |

즉,

- 모든 데이터를 세게 누른 것이 아님
- 딱 한 번 튄 날만 "조금 덜 믿게" 만든 것임

이게 중요하다.  
데이터를 망가뜨린 것이 아니라, **과잉 반응만 줄인 것**이다.

## 7. 왜 이 모델이 좋은지, 중학생도 이해하게 설명

이 모델은 사실 복잡하게 들리지만, 생각은 단순하다.

### 7-1. 최근 시험 성적을 더 중요하게 본다

10월, 11월 점수보다  
1월, 2월 점수가 "지금 실력"에 더 가깝다.

그래서 최근 56일만 본다.

### 7-2. 딱 하루 튄 점수는 조금만 덜 믿는다

평소 5천~7천 건 수준인데  
어느 날만 1만 건이 넘으면,

"이게 평소 실력은 아니겠지"라고 생각해야 한다.

그래서 `2020-01-27` 하루만 살짝 눌렀다.

### 7-3. 요일 패턴은 살린다

온라인 행동 데이터는 요일 차이가 자주 있다.

- 월요일은 높고
- 토요일은 낮고
- 특정 요일은 반복되는 패턴이 있음

ETS는 이런 **7일 주기**를 배울 수 있다.

### 7-4. 추세가 영원히 커진다고 가정하지 않는다

`damped`는 "조금 오르는 흐름이 있더라도 그 기울기가 끝없이 유지되진 않는다"는 뜻이다.

현실적으로는 이게 더 맞다.

즉, 이 모델은 이렇게 생각한다.

> 최근 56일이 중요하고,  
> 요일 패턴은 반복되고,  
> 유독 튄 하루는 조금 덜 믿고,  
> 증가세가 있더라도 무한정 커지지는 않는다.

이 가정이 이번 데이터와 잘 맞았다.

## 10. 그래서 결론이 무엇인가

결론은 명확하다.

1. 기존 전체기간 Holt-Winters는 이 데이터에 맞지 않았다.
   이유는 오래된 급등 구간까지 똑같이 학습했기 때문이다.

2. 이번 데이터는 "전체 역사 평균"보다 "최근 구간 패턴"이 더 중요했다.

3. 아주 강한 전처리가 필요한 것은 아니었다.
   최근 56일 중 **딱 1개의 급등일만 약하게 완화**해도 큰 개선이 나왔다.

4. 최종적으로 가장 좋은 전략은:
   **최근 56일 + 주간 계절성 + damped trend + 상단 1.5% 캡핑**
